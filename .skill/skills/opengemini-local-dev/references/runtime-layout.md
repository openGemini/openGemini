# Runtime Layout

## Core Binaries

- `build/ts-server`: standalone combined server
- `build/ts-meta`: metadata node
- `build/ts-store`: storage node
- `build/ts-sql`: SQL and HTTP entrypoint
- `build/ts-monitor`, `build/ts-data`, `build/ts-recover`: supporting tools

## Config Files

- `config/openGemini.singlenode.conf`: standalone config
- `config/openGemini.conf`: cluster template used by local cluster scripts
- `config/openGemini-*.conf`: generated by `scripts/install_cluster.sh`; treat as disposable

## Single-Node Runtime

Started by `bash scripts/install.sh` or `./build/ts-server run -config=config/openGemini.singlenode.conf`.

Main log path:
- `/tmp/openGemini/logs/server_extra.log`

## Pseudo-Cluster Runtime

Started by `bash scripts/install_cluster.sh`.

Generated log directories:
- `/tmp/openGemini/logs/1`
- `/tmp/openGemini/logs/2`
- `/tmp/openGemini/logs/3`

Common log files:
- `/tmp/openGemini/logs/1/meta_extra1.log`
- `/tmp/openGemini/logs/1/store_extra1.log`
- `/tmp/openGemini/logs/1/sql_extra1.log`
- matching `meta/store` logs under `2` and `3`

Common pid files:
- `/tmp/openGemini/pid/meta1.pid`
- `/tmp/openGemini/pid/meta2.pid`
- `/tmp/openGemini/pid/meta3.pid`
- `/tmp/openGemini/pid/store1.pid`
- `/tmp/openGemini/pid/store2.pid`
- `/tmp/openGemini/pid/store3.pid`
- `/tmp/openGemini/pid/sql1.pid`

## macOS Note

`scripts/install_cluster.sh` uses loopback aliases for `127.0.0.2` and `127.0.0.3`. The script prints the exact `ifconfig lo0 alias ...` commands and warns about GNU `sed` if needed.
